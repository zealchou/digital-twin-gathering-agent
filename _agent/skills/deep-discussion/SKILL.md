# 深度討論技能包 1.0 (deep-discussion)

兩種結構化的深度對話模式。本技能包定義流程和規則，主持人的個性、口吻、台詞由主持人自己的設定檔決定。

| 模式 | 預設主持人 | 可換？ | 核心邏輯 |
|------|----------|--------|---------|
| 辯論模式 | 酒館掌櫃（NPC-004） | 可 | 正反交鋒，拆解論點結構 |
| 深度討論模式 | 茶館老闆（NPC-003） | 可 | 異中求同＋同中求異，往下挖 |

---

## 觸發詞

辯論模式：「辯論」「來場辯論」「正反方討論」「過招」
深度討論模式：「深度討論」「深入聊」「挖深一點」

（通常由村長判斷話題適合後自動觸發，或路人甲主動要求時觸發）

# 模式一：辯論模式

## 對戰組合

| 組合 | 處理方式 |
|------|---------|
| 村民 vs 村民 | 最常見，讀設定檔驅動，直接開始 |
| 村民 vs 客座講師 | 主持人先介紹客座講師背景 |
| 講師 vs 講師 | 主持人介紹雙方背景 |
| 路人甲 vs 村民 | 路人甲用自己的話直接上場 |
| 路人甲 vs 客座講師 | 同上 |

## 核心工作流程

### 第一步：觸發與確認

- 村長判斷或路人甲要求 → 村長請主持人出場接管全場
- 主持人出場前，必須先讀取主持人的設定檔，載入其性格和口吻
- 主持人確認：辯論主題、誰 vs 誰、輪數（預設三回合）
- 如果路人甲沒指定選手，主持人可以根據話題自動建議人選
- 確認完畢直接開始

### 第二步：選手調動

- 用 grep 在 `04 數位分身-個人設定檔/` 搜尋與辯論主題相關的村民
- 讀取設定檔，確認誰適合站哪一方
- 路人甲參戰：路人甲用自己的話直接上場
- 人數限制：每方至少 1 人，最多 5 人

### 第三步：辯論執行（預設三回合）

主持人主導節奏，三回合結構：

1. 第一回合「表態」：各自說明立場和理由
2. 第二回合「交鋒」：針對對方的論點回應、質疑、補充
3. 第三回合「收尾」：回應對方的質疑，看有沒有被說動，或產生共識

逐回合輸出：

辯論採逐回合輸出，不等全部回合結束才顯示：

1. 主持人開場宣告 → 輸出
2. 第一回合所有發言 → 輸出
3. 第二回合所有發言 → 輸出
4. 第三回合所有發言 → 輸出
5. 總結報告 → 獨立輸出

Preview 模式下每次輸出同步推送至 Preview。
單一回合選手過多（每方 3 人以上）時，該回合可再分批輸出。

遵守原則：
- 每位村民的發言必須根據其設定檔，不編造事實
- 超出設定檔範圍的論點，村民要說「這個我不太確定」
- 觸及拒絕清單的話題，村民可以退出辯論
- 路人甲可以要求追加回合，主持人判斷有沒有必要

### 第四步：辯論總結

主持人做總結，切換為客觀分析模式：
- 逐一拆解雙方論點結構
- 指出哪些論點有交手、哪些落空
- 分析誰針對的是前提、誰針對的是結論
- 有沒有真正碰到對方的核心
- 不帶個人好惡，只講結構
- 總結可以長，把該說的說清楚
- 說完總結後，問使用者是否繼續
- 使用者說結束了，主持人才退場

### 第五步：交回村長

主持人確認使用者結束後退場，村長重新接手主持。

# 模式二：深度討論模式

## 設計理念

不是要分出誰對誰錯，而是透過對話往下挖出更深層的東西。

兩個核心原則：

**異中求同：** 觀點相反時，往下挖更底層的交叉點。你們底層其實在乎的是同一件事嗎？

**同中求異：** 觀點一致時，禁止互相附和。放大細微差異 — 你們看起來一樣，但切入角度、前提假設、適用邊界真的一樣嗎？

參考方法論：隱性知識提煉的 QBQ 層次分析（成果層 → 標準層 → 定位層 → 方法層 → 原理層 → 覺察層），每往後跳一層問「為什麼」，前一層的隱性知識就變顯性。

## 參與人數

2-5 人（含路人甲）。

## 參與組合

| 組合 | 處理方式 |
|------|---------|
| 村民 + 村民 | 最常見，讀設定檔驅動 |
| 村民 + 客座講師 | 主持人先介紹客座講師背景 |
| 路人甲 + 村民 | 路人甲用自己的話直接參與 |
| 多人混合 | 最多 5 人，主持人控場 |

## 核心工作流程

### 第一步：觸發與確認

- 村長判斷或路人甲要求 → 村長請茶館老闆出場接管
- 茶館老闆出場前，必須先讀取茶館老闆的設定檔，載入其性格和口吻
- 茶館老闆確認：討論主題、參與者有誰
- 如果路人甲沒指定人選，茶館老闆根據話題自動建議 2-3 人
- 確認完畢直接開始

### 第二步：參與者調動

- 用 grep 在 `04 數位分身-個人設定檔/` 和 `05 NPC客座講師/` 搜尋相關人選
- 讀取設定檔，確認誰對這個話題有東西可以聊
- 路人甲參與：路人甲用自己的話直接上場
- 人數控制：2-5 人，太多人挖不深

### 第三步：深度討論執行

**第一輪「各自表態」：**
每位參與者說出自己對這個主題的看法、經驗或立場。

**主持人判斷路線：**
聽完第一輪後，茶館老闆判斷整體方向：

- 偏異（觀點明顯不同）→ 走「異中求同」路線
- 偏同（觀點大致一致）→ 走「同中求異」路線
- 混合（部分同部分異）→ 兩條路線交替使用

**異中求同路線：**
- 主持人引導：「你們表面上看起來不一樣，但底層在乎的是不是同一件事？」
- 用 QBQ 層次往下挖：從做了什麼（成果）→ 怎麼判斷好不好（標準）→ 這標準反映什麼（定位）→ 為什麼用這標準（方法）→ 為什麼這方法有效（原理）
- 每一輪往下推一層，不急著跳到底
- 目標：找到雙方在更底層其實在乎同一件事的那個交叉點

**同中求異路線：**
- 主持人引導：「你們看起來想法一樣，但我好奇⋯⋯」
- 放大細微差異的五個角度：
  1. 切入角度不同？（從哪裡開始想這件事）
  2. 前提假設不同？（預設了什麼條件才成立）
  3. 適用邊界不同？（什麼情況下這想法會失效）
  4. 優先順序不同？（都同意但覺得哪個更重要）
  5. 來源經驗不同？（用什麼經歷得出這個結論）
- 目標：讓「看起來一樣」的觀點露出真正的差異

**後續輪次：**
- 主持人每輪用一個具體的問題引導，不丟空泛的「你們覺得呢」
- 參與者回應後，主持人整理「剛才我聽到的是⋯⋯」再推進下一層
- 可以在異中求同和同中求異之間切換（挖到交叉點後可能又發現新的差異）

### 第四步：深度討論收尾

茶館老闆整理這場討論挖到了什麼：
- 我們從哪裡開始（表層觀點）
- 挖到了什麼（底層發現）
- 有什麼是一開始沒看到的（新洞察）
- 還有什麼沒挖完的（未解問題）
- 不做勝負判定，不評價誰的觀點「更好」
- 說完整理後，問使用者是否繼續
- 使用者說結束了，茶館老闆才退場

### 第五步：交回村長

茶館老闆確認使用者結束後退場，村長重新接手主持。

# 共用規則

## 輸出格式

使用小聚標準 Markdown 格式（無粗體），例如：

```markdown
## 【主持人名稱】：
開場宣告，說明主題、參與者。

---

## 【參與者A】：
發言內容

---

## 【參與者B】：
回應內容

---

## 【主持人名稱】：
整理剛才聽到的，引導進入下一層。
```

總結格式：

```markdown
## 【主持人名稱】：

（主持人進入整理模式，具體的動作描寫和語氣轉換由主持人設定檔決定）

整理討論脈絡⋯⋯
挖到的底層發現⋯⋯
尚未解決的問題⋯⋯

還要繼續嗎？

（使用者確認結束後，主持人退場，退場方式由設定檔決定）
```

## 行為規則模板

路人甲可以用自然語言設定討論行為，主持人負責監督：

- 頑固型：至少堅持 N 輪不改立場。每輪用不同角度堅持，不是無腦重複同一句話。
- 快速學習型：每輪都有進步。軌跡要求：疑惑 → 半懂 → 嘗試 → 修正。
- 挑戰型：專門找漏洞。不是為反對而反對，而是能提出有深度的合理質疑。
- 被動型：不主動提問，只回答被問到的，用來測試另一方的引導能力。

## 特殊處理與應急機制

1. 沒料了：當村民設定檔不足以支撐後續交鋒或深挖，主持人直接縮短輪數或喊停。
2. 一面倒：辯論模式中如果變成單方面碾壓，主持人幫弱勢方找角度。
3. 中途插花：路人甲想直接參戰，主持人立刻讓他上場。
4. 觸碰紅線：出現違反安全規範或拒絕清單，主持人讓該村民安全退下，並補其他人上。
5. 無限繞圈：當雙方卡在同一觀點重複互咬，主持人推進話題或換一個挖掘角度。
6. 火花四射：出現超有價值的分歧或洞察，主持人允許加碼延長。

## 驗收標準（給 AI 的執行指標）

1. 村民發言是否完全貼合其知識範圍與人設背景。
2. 主持人的口吻是否符合其設定檔（不是寫死在本規則裡）。
3. 辯論模式：三回合結構「表態 → 交鋒 → 收尾」是否有落實。
4. 深度討論模式：是否有往下挖（不停在表層互相附和或重複分歧）。
5. 深度討論模式：異中求同時是否有找到交叉點；同中求異時是否有放大差異。
6. 有準確透過 grep 進行選角。
7. 與村長的交接過程流暢。
