# profile-builder v2.0

## 一，這個技能包在做什麼

專門製作數位分身小聚的角色設定檔。目前主要場景是數位分身小聚村莊，但不排除外來其他應用。

---

## 二，三種角色的分工（重要）

數位分身小聚目前有三種角色，這個技能包只負責其中兩種：

一，村莊管理 NPC：不由本技能包處理
- 這是機制設定，不是角色設定
- 要搭配整體村莊機制技能包另外規劃
- 詳見 `02 世界觀、村規與風格設定`

二，客座講師：本技能包處理
- 從真實講者內容提煉，必須去識別化
- 三層級架構：標準版、副本版、方法論版
- 詳見 `05 NPC客座講師/_客座講師設定檔規格 v1.0.md`

三，村民（真人分身）：本技能包處理
- 一個人可做一到兩個分身
- 使用九宮格結構
- 存放於 `04 數位分身-個人設定檔/`

使用者來建設定檔時，第一件事是確認要做的是哪一種。NPC 的需求要轉出去，不在這裡處理。

---

## 三，核心架構：九宮格 + 三層決策系統

所有設定檔（村民與客座講師）共用九宮格主架構：

一，快速摘要
二，基本資訊
三，價值主張錨點
四，溝通風格
五，思維模式
六，人際互動（含行為彈性）
七，興趣喜好與話題地圖
八，專業能力與工作流程
九，底層邏輯與判斷標準

三層決策系統（又稱思考引擎）（三、五、九三塊合起來）：
- 價值主張錨點（區塊三）＝選擇了什麼（結果）
- 思維模式（區塊五）＝怎麼選擇（動作）
- 底層邏輯（區塊九）＝憑什麼相信這樣選是對的（信念）

完整定義與每個區塊的寫法要點，見 `references/persona-nine-blocks.md`。

---

## 四，安全檢查（每次必做，優先於其他動作）

開始任何設定檔製作前，先讀 `references/safety-checklist.md`，按清單走：

一，前置檢查（用途、角色類型、同意範圍）
二，隱私紅線（絕對不寫進設定檔的八類資訊）
三，禁止出現的內容（連結、攻擊性、代替本人做決定等）
四，客座講師專屬的去識別化規則
五，標準安全聲明（結尾必附）

遇到使用者主動提供隱私資訊時，當場提醒、不收錄。這是硬規則，不接受例外。

---

## 五，三條製作路徑

路徑一：本地對話提煉
使用者在江江的知識庫環境，直接透過對話引導提煉。依 `references/villager-spec.md`（村民）或 `05 NPC客座講師/_客座講師設定檔規格 v1.0.md`（客座講師）執行。

路徑二：NotebookLM 自動化
客座講師設定檔的主要路徑。把講者的內容（影片字幕、文章、逐字稿）丟進 NotebookLM，搭配專用提示詞自動產出草稿，再回本地做去識別化與九宮格整理。完整流程見客座講師規格檔。

路徑三：外部 AI 提示詞包
使用者帶去 ChatGPT、Gemini、Claude 等外部 AI 自行製作。提示詞全文見 `references/path3-prompt-pack.md`。外部成品回來後，仍需走本地的安全檢查與九宮格對照。

## 六，執行流程（使用者說「幫我建數位分身」時）

一，判斷角色類型（村民／客座講師／NPC）
- NPC → 告知不在本技能包範圍，引導去村莊機制技能包
- 村民或客座講師 → 繼續

二，讀 `references/safety-checklist.md`，跟使用者確認前置三件事

三，選路徑（本地對話／NotebookLM／外部 AI）

四，依對應規格執行
- 村民 → `references/villager-spec.md` + `references/persona-nine-blocks.md`
- 客座講師 → `../../../05 NPC客座講師/_客座講師設定檔規格 v1.0.md` + `references/persona-nine-blocks.md`

五，產出後逐條安全檢查

六，結尾附上標準安全聲明

七，依命名規則存檔，並提醒使用者下游運作規範在 `02 世界觀、村規與風格設定/2026-02-05 參考資料-數位分身安全規範摘要.md`
