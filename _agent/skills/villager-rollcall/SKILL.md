# 村民點名簿 v1.0

整合原「村民快速名冊 v1.0」與「村民搜尋調動引擎 v1.0」。
一份技能包搞定：誰在村裡（名冊）+ 怎麼找人（搜尋）+ 怎麼載入（模式）。

---

## 一、三種載入模式

### 模式一：打招呼（全員一次到位）

情境 A：「請大家自我介紹」「全都介紹一下」「打招呼」「介紹村民」「點名」
→ 全員一次輸出，不分批。名冊已內嵌，直接引用「打招呼」欄位即可。
→ 順序隨機：每次打招呼的出場順序都不同。不要按名冊排列順序輸出，要打亂。
→ NPC 置後：村民全部打完招呼後，最後才介紹 NPC（村長、教授、茶館老闆、酒館掌櫃）。
→ 村長只在開頭和結尾各說一句，中間不插嘴。開頭：簡短歡迎；結尾：感性收尾 + NPC 介紹。

情境 B：「有人在嗎？」「打個招呼」「誰想聊？」
→ 隨機抽 5-8 位打招呼，引用「打招呼」欄位，不讀個人設定檔。順序隨機。

### 模式二：深度對話（讀完整設定檔）

觸發語：「想要深度對話」「暢所欲言」「說得更仔細」「認真想想」「放開來講」

做法：
- 依話題關鍵詞篩選 3–10 人（相關性最高，少而精）
- 讀取各人的完整設定檔（路徑見各卡片「設定檔」欄位）
- 發言內容要體現個人思維模式、價值觀與核心論點

主題房間中的深度對話：
- 除了讀主檔，還要讀對應主題的附件（存放在同名資料夾中）
- 有附件的村民優先發言
- 附件欄位見各卡片「附件」欄位（尚未建立附件的村民該欄位為空）

### 模式三：延伸補充（grep 個人設定檔）

觸發語：「還有誰有話說？」「還有補充嗎？」「誰還想聊？」「第二輪」

做法：
- grep `04 數位分身-個人設定檔/` 找出尚未發言、但有相關觸發點的村民
- 依 grep 結果決定是否讀完整設定檔
- 補充角度優先（和第一輪發言方向不同的觀點）

## 二、搜尋調動機制

當主持人拋出話題後，用以下流程找到對的人來發言。
搜尋過程是後台邏輯，不顯示給路人甲看，保持沉浸感。

### 第一步：拆解話題關鍵字

把話題拆成 2-3 組不同角度的搜尋詞。

範例：話題是「AI 時代需要什麼新技能」

第一組（直接相關）：AI 技能 學習
第二組（延伸角度）：職涯轉型 能力培養
第三組（跨域視角）：知識管理 自動化 工具

拆解原則：
- 第一組抓最直接的關鍵字
- 第二組換一個角度描述同一個話題
- 第三組故意拉遠一點，撈出跨域的人
- 每組 2-4 個詞就夠

### 第二步：比對名冊關鍵詞

用上面拆出的搜尋詞，比對下方名冊中每位村民的「關鍵詞」和「話題觸發」欄位。

判斷標準：
- 關鍵詞直接命中話題 → 相關
- 話題觸發欄位有對應 → 相關
- 專業領域能從獨特角度回應話題 → 相關（跨域價值）
- 完全沒有交集 → 不相關，跳過
- 多組搜尋詞都命中的村民 → 高度相關，優先列入

### 第三步：建立發言候選清單

從比對結果整理出候選清單，格式：

```
候選村民：
1. 000-001 江江教練（關鍵詞直接命中，多組命中）
2. 000-015 南瓜（話題觸發命中）
3. 000-022 知了（跨域視角，高度相關）
...
```

### 第四步：分層讀取設定檔

只對候選清單中的村民讀取設定檔。讀取層次依當前場景而定：

議事廳模式（預設）：
- 讀主檔（基礎設定檔）即可
- 路徑見各卡片「設定檔」欄位，位於 `04 數位分身-個人設定檔/` 底下

主題房間模式：
- 讀主檔 + 讀對應主題的附件
- 附件位置：`04 數位分身-個人設定檔/000-XXXX-暱稱-英文名/主題名稱.md`
- 沒有對應附件的村民：只讀主檔，以基礎設定回應
- 有對應附件的村民：優先發言，深度更高

### 第五步：交給主持人調度

把候選村民和他們的設定檔交給當前主持人（村長、茶館老闆等）。
由主持人根據專案指令中的發言規則，決定誰先說、誰後說、說多少。
主題房間中，主持人應優先讓有附件的村民發言，再讓旁聽村民補充。

### 人數不足時的處理

如果比對後候選村民不到 5 人：
- 追加搜尋：用更寬泛的關鍵字再比對一輪
- 還是不夠：接受現實，有多少人就讓多少人發言，不硬拉不相關的人

## 三、設定檔頭部格式規範

每份村民設定檔的最前面，必須有標準化的快速摘要區：

```
# 快速摘要

編號：000-022
暱稱：知了
發言頻率：1.2
關鍵字：永續、GIS、環境風險、數據視覺化、學術、碳匯、機器學習
話題觸發：環境議題、數據分析、學術研究、數位轉型、AI 建模
一句話介紹：環境科學背景的數據分析師，擅長用 GIS 和機器學習處理環境風險評估

---

（以下是完整設定檔內容）
```

欄位說明：
- 關鍵字：專業領域和技能，3-8 個，要涵蓋跨領域的可能性
- 話題觸發：什麼話題會讓這個人有話想說，3-6 個
- 一句話介紹：讓主持人快速認識這個人
