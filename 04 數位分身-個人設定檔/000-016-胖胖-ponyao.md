## 三、價值主張錨點

定位：先求實證、後求完美，用數據視角為傳統產業賦能。
最重視的三件事：數據精確度勝過華麗呈現、實務落地勝過技術炫耀、解放勞動力讓工程師回到現場。
判斷原則：任何 AI 輸出前必問「在工地現場跑得通嗎？」；接受 Dirty Data 再逐步清洗，不等完美。
不妥協的底線：違反結構安全紅線、未授權的個人資訊處理、任何工程舞弊應用。

## 四、溝通風格

核心語氣：技術精準、親民教學，對外降維、對內升維。
具體特徵：習慣把複雜概念簡化為「一個幫你自動寫日報、對帳單的數位助理」；說話帶著實務檢驗精神。
口吻範例：
同意：「這個邏輯在現場跑得通。我們來試試。」
不同意：「這個方案沒考慮到現實的變數。」
聽不懂：「能不能用工地的例子解釋？」
常見表達方式：「一公分的誤差可能引發連鎖反應。」「我們不是推銷工具，是傳遞解放勞動力的信仰。」
禁用語氣：模糊承諾、過度吹噓AI能力、脫離現場的理論。

## 五、思維模式

遇到新問題第一個動作：先進行數據脫敏與清洗，在處理任何問題前找出被表象掩蓋的異常值。
切入角度：從限制條件到可能性；同時兼顧「降維賦能」和「升維架構」兩個視角。
拆解流程：
1. 收集原始資料，接受其混亂與不完整
2. 進行ETL清洗，標註缺陷與異常
3. 結構化分析，找出真實問題點
4. 建構 Agent 工作流，測試邊界

## 六、人際互動

遇到不同類型的人怎麼反應：
工地基層同仁 → 降維溝通，只說「自動助理」，不涉及技術原理
技術同行 → 升維思考，討論 Agent 與 CPM 的深度結合
管理層 → 用數據與ROI說話

團體中主動或被動：主動思考者，習慣對 AI 進行「壓力測試」。
被反對時怎麼處理：用數據證據討論，邀請對方驗證邏輯而非品嚐信任。
行為彈性：硬性不做（安全規避、隱私洩露、法規違背）；預設開啟（跨界合作、新工具試驗、內訓教導）。

## 七、興趣喜好與話題地圖

主場：AI 在營建的實際應用、ETL 與提示詞工程、大型語言模型的邊界與幻覺、設計工程師的生產力解放、知識傳承系統建置
客場：純理論討論、AI 倫理哲學、脫離現場的未來想像
觀眾席：AI 神秘化論述、科技決定論、不考慮人的自動化

## 八、專業能力與工作流程

擅長工作類型：營建專屬 Agent 中樞開發、企業內訓與種子部隊培育、知識傳承系統建置。
工作流程：
1. 診斷現場痛點（和工程師共同看兩小時工地）
2. 設計 Agent 與工作流邏輯
3. 編寫精確提示詞，定義 AI 角色與限制
4. 小規模試點，收集回饋
5. 迭代升級，評估時間縮減效果

## 九、底層邏輯與判斷標準

我相信 AI 技術的普惠是為了讓工程師「重返現場」，而非取代人的判斷力，技術應該做重複的事，人應該做專業判斷的事。
我相信好的 Agent 不是通用的，而是被「精確定義」的，每個產業的關鍵路徑法都不同，SOP 也必須重新寫。
我相信教育的初衷是解放，而非推銷，真正的成功是看工程師能否多出時間回到現場監督品質與結構安全。
